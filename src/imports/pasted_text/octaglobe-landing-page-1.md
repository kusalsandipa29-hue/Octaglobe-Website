# Figma Make Prompt — OctaGlobe Landing Page

Design and prototype a premium landing page for **OctaGlobe**, a software product R&D company that engineers connected operational systems for logistics and enterprise organizations.

The final result should feel engineered, structured, and minimal. Every section should have a clear purpose. Avoid decorative elements that do not support the content.

---

## Overall Design Direction

The interface should communicate precision, structure, and technical credibility rather than marketing.

The experience should feel comparable in quality to companies like Stripe, Linear, Vercel, Notion, and Figma without imitating their layouts or visual language.

The design must prioritize readability, hierarchy, spacing, and consistency over visual complexity.

---

## Brand Personality

Design should feel:

* Modern
* Premium
* Technical
* Minimal
* Structured
* Confident
* Intelligent
* Calm
* Purposeful

Avoid:

* Glassmorphism
* Excessive gradients
* Neon effects
* Busy backgrounds
* Large decorative illustrations
* Generic SaaS artwork
* Marketing-style graphics
* Cartoon icons
* Heavy shadows

---

## Brand Colors

Primary Dark: **#141228**

Signal Green: **#03EF62**

White: **#FFFFFF**

Color usage should follow the brand system.

* Dark is the structural color.
* Green is used as a signal and accent.
* White provides clarity and separation.

Green should only highlight important actions, active states, key interface elements, and small accents.

Never flood sections with green.

---

## Typography

Use a modern geometric sans-serif.

Recommended:

* Outfit
* Inter
* Geist

Large, bold headlines.

Comfortable body text.

Generous line spacing.

Strong visual hierarchy.

## Layout

Maximum content width:
**1280px**

Responsive 12-column grid.

Large whitespace between sections.

Consistent vertical rhythm.

8px spacing system.

Use cards only when they improve organization.

## Hero Section

This is the visual centerpiece.

Do not use a static hero.

Instead, create an interactive experience.

### Left Side

Large headline.

Supporting paragraph.

Primary CTA.

Secondary CTA.

### Right Side

Create an animated visualization representing connected operational systems.

Possible approaches:

* Modular nodes connecting together
* System architecture diagram
* Connected workflow blocks
* Operational network visualization
* Interactive dashboard preview
* Dynamic logistics map abstraction
* Modular software ecosystem

The animation should continuously evolve rather than loop obviously.

Connections should expand and reorganize naturally.

Hovering elements should reveal additional information.

Movement should be subtle.

No floating random shapes.

No meaningless particles.

---

## Hero Background

Use a minimal grid.

Very subtle radial lighting.

Low-opacity architectural lines.

Very light animated pattern.

Never distract from content.

---

## Scroll Experience

Every section should animate into view.

Animation style:

* Fade + translate Y
* 300–500ms duration
* Slight stagger between child elements
* Smooth easing

Avoid:

* Bounce
* Elastic
* Zoom
* Spin
* Flip

Scrolling should feel effortless.

---

## Company Introduction

The other visualizes the relationship between disconnected tools and a connected operational system.

Use clean diagrams rather than illustrations.

## Motion Design

Motion should reinforce structure.

Everything should feel engineered.

Recommended interactions:

* Smooth hover transitions
* Animated underline
* Progressive reveal
* Connected node animations
* Timeline progression
* Cursor-aware hero visualization
* Soft parallax on background grid
* Card elevation
* Animated CTA highlight

Avoid attention-seeking animations.

---

## Performance

Design for fast loading.

Prefer CSS-based animations.

Minimize heavy effects.

Avoid large videos.

Avoid unnecessary Lottie files.

Use lightweight SVG graphics wherever possible.

Design should feel responsive even on slower devices.

---

## Accessibility

Maintain high contrast.

Large clickable areas.

Readable typography.

Keyboard-friendly navigation.

Respect reduced motion preferences by disabling non-essential animations.

---

## Design Principles

Every visual element should communicate structure.

Every animation should have a purpose.

Every interaction should reinforce the idea of connected systems.

Remove anything that feels decorative without adding meaning.

The finished landing page should communicate that OctaGlobe is an engineering-led software company that builds connected operational platforms through modular architecture, research-driven product development, and long-term system thinking.
